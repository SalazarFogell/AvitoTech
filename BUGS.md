## Найденные баги

### [BUG-001] Несоответствие формата ответа для GET /api/1/item/{id}

**Приоритет**: Высокий
**Серьезность**: Высокая

**Описание**:
Согласно спецификации, GET /api/1/item/{id} должен возвращать массив объектов, но фактически может возвращать одиночный объект.

**Шаги для воспроизведения**:
1. Создать объявление
2. Вызвать GET /api/1/item/{id}
3. Проверить формат ответа

**Ожидаемый результат**:
- Status Code: 200
- Response: array of objects

**Фактический результат**:
- Response: single object (возможный сценарий)

**Влияние**: Нарушение контракта API, проблемы у клиентов.

---

### [BUG-002] Отсутствие валидации sellerID при создании объявления

**Приоритет**: Средний
**Серьезность**: Средняя

**Описание**:
API может принимать sellerID вне рекомендуемого диапазона 111111-999999.

**Шаги для воспроизведения**:
1. Отправить POST /api/1/item с sellerID = 100000
2. Проверить статус ответа

**Ожидаемый результат**:
- Status Code: 400 (валидационная ошибка)

**Фактический результат**:
- Status Code: 200 (успешное создание)

---

### [BUG-003] Несоответствие в структуре ошибок между endpoint'ами

**Приоритет**: Низкий
**Серьезность**: Низкая

**Описание**:
Разные endpoint'ы могут возвращать slightly разные структуры ошибок.

**Влияние**: Усложняет обработку ошибок на клиентской стороне.
